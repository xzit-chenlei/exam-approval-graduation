# 参与贡献

感谢你对本项目的关注。本文说明如何参与「考试审批与毕业达成度系统」的开发与协作。

## 许可协议

本仓库以 [Apache License 2.0](LICENSE) 授权。你向本仓库提交的代码、文档或其他材料，将视为在相同许可证下贡献；若你不同意，请勿提交。

## 报告问题

- 先在 Issues 中搜索是否已有相同或相关问题。
- 新建 Issue 时请尽量说明：环境（JDK、MySQL、Redis 版本等）、复现步骤、期望行为与实际行为；若是缺陷，可附上日志或截图（注意脱敏）。

## 提交变更（Pull Request）

1. **Fork** 本仓库，从你 Fork 的分支上新建功能分支（命名建议：`feat/简述`、`fix/简述`）。
2. 本地开发与自测：至少完成与你改动相关的编译与基本验证（例如 `mvn -pl xzit-starter -am package -DskipTests` 或通过现有测试）。
3. **保持变更聚焦**：一次 PR 解决一类问题，避免无关格式化或与需求无关的大范围重构。
4. 提交信息使用清晰的中文或英文说明「做了什么、为什么」，便于审阅与回溯。
5. 发起 PR 时简述动机、主要改动点；若关联 Issue，请写上编号。

## 开发环境（摘要）

更完整的步骤见仓库根目录 [README.md](README.md)。

- JDK 8 或 17，Maven 3.6+
- MySQL、Redis 按 README 配置并导入基线 SQL
- 敏感配置（数据库密码、阿里云密钥等）勿提交仓库，优先使用环境变量或本地未跟踪的配置

## 代码风格

- 与现有模块的包结构、命名与注释风格保持一致。
- 新增逻辑优先复用已有工具类与服务，避免重复实现。
- 勿在提交中包含真实密钥、内网地址或个人隐私数据。

## 审阅说明

维护者会尽快审阅 PR；如需修改，会在 PR 中留言。感谢你的耐心与贡献。
