# 图表中文乱码修复部署文档

## 1. 问题根本原因
Windows本地生成图表中文完全正常，
**WSL / Linux 服务器 生成图片中文乱码、方框**，
核心原因：
代码硬编码依赖 Windows 专属的宋体/新宋体字体，Linux 默认未预装该字体。

## 2. 必须安装的字体清单
| 中文名称 | 系统字体名 | 字体样式 |
|---|---|---|
| 宋体 | SimSun | Regular（常规） |
| 新宋体 | NSimSun | Regular（常规） |

---

## 3. WSL 环境一键安装字体
WSL 可以直接复用本机Windows字体，无需额外下载：
```bash
# 更新软件源、安装字体工具
sudo apt update && sudo apt install -y fontconfig

# 创建系统字体目录
sudo mkdir -p /usr/share/fonts/windows-chinese

# 直接从Windows挂载目录复制宋体、新宋体
sudo cp /mnt/c/Windows/Fonts/simsun.ttc /usr/share/fonts/windows-chinese/

# 刷新全局字体缓存
sudo fc-cache -fv

# 验证是否安装成功
fc-list :lang=zh | grep -E "SimSun|NSimSun"
