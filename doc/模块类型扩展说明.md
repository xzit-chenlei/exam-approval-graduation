# 模块类型扩展说明

## 概述
为考试模块系统添加了三种模块类型支持：
- **类型1（数据类型）**：二维数据列表（原有功能）
- **类型2（文件类型）**：文件上传
- **类型3（日期选择）**：日期选择

## 数据库修改

### 1. SQL迁移脚本
文件：`exam-flow-back/doc/update_module_type.sql`

```sql
ALTER TABLE `exam_module` 
ADD COLUMN `type` int(11) NOT NULL DEFAULT 1 COMMENT '模块类型：1-数据类型，2-文件类型，3-日期选择' AFTER `name`;

UPDATE `exam_module` SET `type` = 1 WHERE `type` IS NULL OR `type` = 0;
```

### 2. 表结构说明
- `type` 字段：模块类型，1-数据类型，2-文件类型，3-日期选择
- `row_headers` 和 `col_headers`：仅类型1使用
- `data` 字段：
  - 类型1：存储二维数组 JSON（`[[...], [...]]`）
  - 类型2：存储文件对象数组 JSON（`[{label, name, url}, ...]`）
  - 类型3：存储日期对象数组 JSON（`[{label, value}, ...]`）

## 后端代码修改

### 1. 实体类修改
**文件**：`exam-flow-back/xzit-core/src/main/java/edu/xzit/core/dao/domain/ExamModule.java`

添加字段：
```java
/**
 * 模块类型：1-数据类型，2-文件类型，3-日期选择
 */
@Excel(name = "模块类型")
private Integer type;
```

### 2. 服务层修改
**文件**：`exam-flow-back/xzit-core/src/main/java/edu/xzit/core/service/FormModuleService.java`

#### 2.1 数据解析（`buildModuleContent`方法）
- 根据 `type` 字段判断模块类型
- 类型1：解析 `rowHeaders`、`colHeaders` 和二维数组 `data`
- 类型2和3：解析对象数组 `data`

#### 2.2 数据保存（`saveFormModules`方法）
- 保存模块类型到 `type` 字段
- 根据类型处理不同的数据字段：
  - 类型1：保存 `rowHeaders`、`colHeaders`、`data`
  - 类型2和3：只保存 `data`，`rowHeaders` 和 `colHeaders` 设为 null

#### 2.3 新增方法
- `parseObjectList(String json)`：解析对象数组，用于文件类型和日期类型

## 前端代码修改

### 1. 模板修改
- 添加模块类型选择器（下拉框）
- 根据类型显示不同的内容区域：
  - 类型1：显示二维数据表格
  - 类型2：显示文件上传组件
  - 类型3：显示日期选择组件

### 2. 数据结构
- 添加 `type` 字段到模块数据
- 文件类型：`data` 为 `[{label, name, url}]`
- 日期类型：`data` 为 `[{label, value}]`

### 3. 核心方法
- `normalizeModule`：支持三种类型的数据解析
- `addModule`：添加模块时选择类型
- `updateModuleType`：切换类型时重置数据结构
- `saveFormData`：根据类型构建不同的保存数据结构

## JSON数据结构

### 保存时的数据结构
```json
{
  "formId": 1,
  "formName": "表单名称",
  "modules": [
    {
      "id": 123,
      "name": "模块1",
      "type": 1,
      "rowHeaders": ["行1", "行2"],
      "colHeaders": ["列1", "列2"],
      "data": [["值1", "值2"], ["值3", "值4"]]
    },
    {
      "id": 124,
      "name": "模块2",
      "type": 2,
      "data": [
        {
          "label": "文件标签",
          "name": "文件名.pdf",
          "url": "文件URL"
        }
      ]
    },
    {
      "id": 125,
      "name": "模块3",
      "type": 3,
      "data": [
        {
          "label": "日期字段名称",
          "value": "2019/05/07"
        }
      ]
    }
  ]
}
```

### 返回时的数据结构
与保存时相同，但包含后端生成的 `id` 字段。

## 兼容性说明

1. **旧数据兼容**：现有数据如果没有 `type` 字段，默认为类型1（数据类型）
2. **字段兼容**：类型2和3的模块，`rowHeaders` 和 `colHeaders` 字段为 null，不影响功能

## 部署步骤

1. **执行SQL迁移脚本**
   ```bash
   mysql -u用户名 -p数据库名 < exam-flow-back/doc/update_module_type.sql
   ```

2. **重新编译后端代码**
   ```bash
   cd exam-flow-back
   mvn clean install
   ```

3. **重启后端服务**

4. **前端代码已更新**，无需额外操作

## 注意事项

1. 文件上传功能需要配置正确的上传接口地址（前端代码中的 `uploadAction`）
2. 日期格式统一使用 `yyyy/MM/dd` 格式
3. 文件上传响应格式需要包含 `url` 或 `data` 字段
